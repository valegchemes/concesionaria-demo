# Setup Verification Checklist

After creating this professional-grade SaaS foundation, verify everything is working correctly.

## ✅ Prerequisites (Must Have)
- [ ] Node.js 20+ installed (`node --version`)
- [ ] PostgreSQL 15+ running (local or Neon cloud)
- [ ] npm or yarn installed

## ✅ Step 1: Install Dependencies

```bash
npm install
```

Verify:
- No error messages
- `node_modules` folder created
- `package-lock.json` updated

**Expected output:**
```
up to date, audited X packages in Y seconds
```

## ✅ Step 2: Generate Prisma Client

```bash
npm run db:generate
```

Verify:
- `node_modules/.prisma/client` generated
- No TypeScript errors

**Expected output:**
```
✔ Generated Prisma Client (X.Y.Z) to ./node_modules/.prisma/client
```

## ✅ Step 3: Verify Environment Variables

```bash
# Copy the example file
cp .env.example .env.local

# Edit .env.local with your PostgreSQL connection string
# Example: postgresql://user:password@localhost:5432/consencionaria
```

Verify:
- `.env.local` file exists
- `DATABASE_URL` is set to your PostgreSQL connection
- `NEXTAUTH_SECRET` is set (or will be generated by Next.js)

## ✅ Step 4: Create Database Migrations

```bash
# This creates migration files for the new schema
npm run db:migrate
```

Verify:
- No errors
- New migration file created in `prisma/migrations/`
- Schema applied to database

**Expected output:**
```
✔ Your database is now in sync with your schema. Commit your migration to version control.
```

## ✅ Step 5: Verify Shared Libraries

```bash
npx tsc --noEmit
```

Verify:
- No TypeScript errors
- All new files compile correctly

**Expected files:**
- ✅ `lib/shared/config.ts` - Environment validation
- ✅ `lib/shared/validation.ts` - Zod schemas
- ✅ `lib/shared/logger.ts` - Pino logging
- ✅ `lib/shared/errors.ts` - Custom error classes
- ✅ `lib/shared/api-response.ts` - API response formatting
- ✅ `lib/shared/auth-helpers.ts` - Auth utilities
- ✅ `lib/shared/prisma.ts` - Prisma client
- ✅ `lib/domains/leads/types.ts` - Lead domain types
- ✅ `lib/domains/leads/service.ts` - Lead service

## ✅ Step 9: Test API Response Formatting

```bash
# In another terminal
curl http://localhost:3000/api/v1/leads \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json"
```

Verify:
- Response has `{ success: true, data: [...], meta: { timestamp } }` format
- Proper HTTP status codes (200, 400, 401, 404, 500)
- Error responses have `{ success: false, error: {...} }` format

## ✅ Step 10: Test Error Handling

Send invalid request:
```bash
curl -X POST http://localhost:3000/api/v1/leads \
  -H "Content-Type: application/json" \
  -d '{"name": ""}'  # Invalid: empty name
```

Verify:
- Returns 400 status (Bad Request)
- Response includes validation errors
- Structure: `{ success: false, error: { message, code, details } }`

## ✅ Step 11: Verify Security Headers

```bash
curl -i http://localhost:3000 | grep -i "x-content-type-options\|x-frame-options\|strict-transport-security"
```

Verify:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`

## 🚨 Common Issues & Fixes

### Issue: "DATABASE_URL is required"
**Fix**: Ensure `.env.local` has `DATABASE_URL` set
```bash
echo "DATABASE_URL=postgresql://user:password@localhost:5432/consencionaria" >> .env.local
```

### Issue: "Port 3000 already in use"
**Fix**: Use different port
```bash
npm run dev -- -p 3001
```

### Issue: "Prisma client not generated"
**Fix**: Regenerate
```bash
npm run db:generate
```

### Issue: "TypeScript errors"
**Fix**: Ensure types are correct
```bash
npm run db:generate  # Regenerate Prisma types
npx tsc --noEmit    # Check all types
```

### Issue: "Database migration failed"
**Fix**: Reset database (development only)
```bash
npm run db:reset
```
